# 创作工具与生产流程

## Purpose

提供网文创作的工具链与生产流程设计，帮助作者高效利用AI辅助工具，建立可持续的写作工作流，实现从构思到发布的全流程优化。

## When to Use

- 用户需要选择AI写作工具
- 用户需要设计提示词模板
- 用户需要规划写作工作流
- 用户需要选择码字/管理工具
- 用户需要制定发布与运营策略

## Instructions

### AI辅助写作工具对比

#### DeepSeek
- 优势：中文理解力强、长文本处理好、性价比高
- 适用：大纲生成、章节初稿、角色对话
- 提示词：需要详细的人设和剧情指令

#### ChatGPT
- 优势：创意丰富、逻辑清晰、多语言
- 适用：创意发散、剧情推演、世界观构建
- 提示词：英文提示词效果更佳

#### Claude
- 优势：长文本处理优秀、风格模仿能力强
- 适用：长篇续写、风格迁移、深度润色
- 提示词：需要提供风格样本

#### 豆包/Kimi
- 优势：中文语境理解好、免费额度多
- 适用：日常写作辅助、快速生成
- 提示词：简洁直接即可

### 提示词模板

#### 大纲生成提示词
```
你是一位专业的网文编辑。请根据以下信息生成一份详细的小说大纲：

题材：[题材类型]
核心设定：[金手指/核心设定]
主角人设：[性格标签+背景]
目标平台：[平台名称]
预计字数：[总字数]

要求：
1. 包含完整的起承转合
2. 每卷有明确的核心冲突
3. 标注爽点分布位置
4. 标注断章钩子位置
5. 字数分配合理
```

#### 章节生成提示词
```
你是一位网文写手。请根据以下信息撰写第[N]章：

前情提要：[前一章概要]
本章目标：[本章要完成的剧情目标]
出场角色：[角色名+当前状态]
爽点设计：[本章爽点类型+结构]
断章钩子：[本章结尾钩子类型]
文风要求：[文风类型]
字数要求：[字数范围]

要求：
1. 开头承接上一章
2. 节奏紧凑，不注水
3. 爽点结构完整
4. 结尾设置钩子
5. 角色行为符合人设
```

#### 角色对话提示词
```
你是一位网文写手。请为以下场景撰写对话：

角色A：[名字]+[性格标签]+[当前情绪]+[说话风格]
角色B：[名字]+[性格标签]+[当前情绪]+[说话风格]
场景：[场景描述]
对话目的：[对话要推动什么剧情]
潜台词：[表面之下真正在说什么]

要求：
1. 每个角色有独特的说话方式
2. 对话自然，不生硬
3. 包含潜台词
4. 推动剧情发展
5. 适当使用语气词
```

### 写作工作流设计

#### 标准工作流

1. **构思阶段**（1-2天）
   - 确定题材与核心设定
   - 生成大纲框架
   - 设计主要角色

2. **大纲阶段**（3-5天）
   - 细化大纲至章节级
   - 标注爽点与钩子位置
   - 建立设定文档

3. **开篇阶段**（1周）
   - 精写黄金三章
   - 反复打磨开篇
   - 内投签约

4. **连载阶段**（持续）
   - 日更2-3章
   - AI辅助+人工改写
   - 保持存稿3-5天

5. **运营阶段**（持续）
   - 关注数据反馈
   - 与读者互动
   - 调整写作策略

### 码字与管理工具

- **码字精灵**：专注码字，番茄钟+字数统计
- **墨者写作**：多端同步，云端存储
- **橙瓜码字**：网文专用，数据统计
- **石墨文档**：协作写作，多端同步
- **Notion/Obsidian**：设定管理，知识库

### 发布与数据分析

#### 关键数据指标
- 追读率：读者持续阅读的比例
- 完读率：读完单章的比例
- 收藏率：阅读后收藏的比例
- 推荐票/月票：读者认可度
- 评论活跃度：读者参与度

#### 数据优化策略
- 追读率低→优化开篇和断章
- 完读率低→减少注水，加快节奏
- 收藏率低→增强钩子和期待感
- 评论少→增加互动话题点
