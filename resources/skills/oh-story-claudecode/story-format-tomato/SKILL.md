# 番茄小说平台爽文排版格式化

## 核心定位

针对番茄小说等移动端平台的"爽文"排版优化系统。核心目标：让读者在手机屏幕上获得最佳阅读节奏体验。

## 排版核心规则

### 段落控制（最高优先级）

1. **每段最多80字或2句话**（取先到者）
2. **对话必须独立成段**，前后各留空行
3. **动作描写独立成段**，与对话分离
4. **心理活动独立成段**，用斜体或无引号区分
5. **禁止超过3行的段落**（手机屏幕标准）

### 对话格式

```
"对话内容。"角色说道。

→ 改为：

"对话内容。"

角色说道。
```

- 对话标签（他说/她道）独立成段或省略
- 连续对话每轮独立成段，中间插入1句动作/表情

### 节奏加速器

#### 短句密集区（高潮/打斗/追逐）
- 每句≤15字
- 每段仅1句
- 连续5-10个单句段制造紧迫感

#### 情绪外化规则
- 禁止"他感到愤怒" → 改为具体生理反应
- 禁止"她很伤心" → 改为动作/环境投射
- 所有情绪必须通过外在表现传达

### 视觉化动作词替换字典

| 原词 | 替换为 |
|------|--------|
| 看 | 盯/瞥/扫/瞄/凝视 |
| 走 | 迈步/踱步/疾走/挪动 |
| 说 | 低声道/冷笑道/沉声道 |
| 想 | 脑海中闪过/意识到 |
| 快 | 电光火石间/刹那间 |

## 人性化润色规则集（v2.0）

### AI内容删除规则

以下内容必须直接删除，不做替换：
- "值得注意的是"、"需要指出的是"、"众所周知"
- "综上所述"、"总而言之"、"不难发现"
- "首先...其次...最后..."结构（打散为自然叙述）
- "一方面...另一方面..."结构
- 所有元叙述（"本章将讲述..."、"接下来我们看到..."）

### 格式规范化

| 项目 | 规则 |
|------|------|
| 引号 | 统一为「」（对话）和『』（引用） |
| 省略号 | 统一为……（两个中文省略号） |
| 破折号 | 统一为——（两个em dash） |
| 数字 | 年份→中文（一九八七年）；数量→阿拉伯（3个） |
| 感叹号 | 每章≤5个，禁止连续使用（!!!） |

## LM重写指令

对每个段落执行以下检查：
1. 是否超过80字？→ 拆分
2. 是否包含AI禁用词？→ 删除或替换
3. 对话是否独立成段？→ 分离
4. 情绪是否直接告知？→ 改为外化表现
5. 动作词是否平庸？→ 查替换字典
6. 是否有连续3段以上无对话？→ 插入对话打断

## 执行优先级

1. 段落拆分（结构层）
2. AI痕迹清除（内容层）
3. 格式规范化（标点层）
4. 词汇替换（词汇层）
5. 节奏优化（节奏层）
