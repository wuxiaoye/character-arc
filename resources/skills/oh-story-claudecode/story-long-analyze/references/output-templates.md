## 速查参考

### 爽点类型速查

| 类型 | 定义 | 经典例子 |
|------|------|----------|
| 装逼打脸 | 实力碾压被轻视的人 | 「你也配？」 |
| 逆袭反转 | 绝境翻盘 | 废材突然觉醒 |
| 获得机缘 | 得到别人没有的东西 | 捡到神器/奇遇 |
| 信息差碾压 | 读者和主角知道别人不知道的 | 重生者的先知优势 |
| 情感满足 | 人物关系的发展 | 告白、团聚、认可 |
| 扮猪吃虎 | 隐藏实力突然展露 | 扮演弱者实则是强者 |

### 钩子类型速查

| 类型 | 定义 | 示例 |
|------|------|------|
| 悬念钩 | 抛出未解之谜 | 「这把剑为什么会认他为主？」 |
| 冲突钩 | 直接展示对抗 | 开篇就是追杀 |
| 反差钩 | 违反常识的信息 | 「穿越第一天就破产了」 |
| 代入钩 | 让读者产生共鸣 | 和读者处境相似的困境 |
| 信息差钩 | 给读者角色不知道的信息 | 「他还不知道，站在他面前的是……」 |

### 爽点循环结构速查

爽文章节内部运行的三层微观循环：

**铺垫层（蓄势）**：通过压抑、困境、信息差制造情绪缺口。读者带着"他到底会怎么做"跟进。
**释放层（爽感）**：主角展露实力/打脸/碾压。爽感强度由反应层数和篇幅决定，而非装逼本身。
**衔接层（承转）**：旧循环结束埋下新钩子（身份反转、规模升级、新威胁），读者满足后立刻进入下一期待。

反应层递进模型（决定释放层质量）：

```
质疑/轻视 → 震惊/打脸 → 强者重新评价 → 扩散效应
```

反应篇幅比 = 反应总字数 ÷ 装逼字数。低于1.5说明装逼写完就翻篇了，读者等来的爽感没被接住。

诊断规则：某章"主角很厉害但读着不爽"时，检查反应层数是否足够。修复方法：不改装逼场景，在后面补2-3层外部反应即可。

### 衔接过渡模式速查

衔接层不是一个独立的新钩子，而是一个**双功能节点**——同时完成当前循环的收尾和下一循环的开端。

5 种典型过渡模式：

| 模式 | 机制 | 示例 |
|------|------|------|
| 打完小的来大的 | 当前释放层的对手引出更强对手 | 打败弟子→师父找上门 |
| 胜利中发现隐患 | 释放成功但暴露新问题 | 赢了比赛→发现中毒 |
| 身份暴露/反转 | 释放过程中身份被识破，危机升级 | 打脸后被人认出真实身份 |
| 利益升级 | 释放获得的成果指向更大目标 | 拿到线索→指向更深层宝藏 |
| 关系突变 | 释放改变人物关系格局，触发新冲突 | 救了的人变成仇家后代 |

诊断规则：某章"爽完就断、下章接不上"时，检查衔接层是否缺失。修复方法：在释放尾声补一个双功能节点即可。

### 题材爽点形态速查

不同题材的"释放层"形态差异大，分析时需用对应视角：

| 题材 | 释放层典型形态 | 反应层核心来源 | 铺垫层常见缺口制造 |
|------|----------------|----------------|---------------------|
| 玄幻/修仙 | 实力碾压、境界突破 | 同门/对手/长老震惊 | 功法残缺、资源匮乏、血脉压制 |
| 都市 | 预测准确、身份揭露 | 商业对手/前女友/权贵重评 | 破产/被辞/被退婚 |
| 系统文 | 解锁稀有技能、任务奖励 | 系统提示的稀有度标注 | 任务失败惩罚、技能冷却 |
| 历史 | 以弱胜强、先知优势 | 谋士/帝王/敌将震惊 | 信息不对称、身份低微 |
| 悬疑 | 真相揭露、反杀 | 被揭穿者的崩溃/读者的恍然 | 线索误导、嫌疑人洗白 |
| 游戏/诸天 | 碾压副本、跨级击杀 | 队友/对手/直播间观众 | 资源不足、队友拖后腿 |

使用规则：分析前先查此表确定题材的释放形态，再套用循环模板。混合题材取主成分。

### 深度 阶段2：角色过滤规则

| 分类 | 标准 | 提取深度 |
|------|------|----------|
| 不提取（路人） | 只出场1次且无台词，纯背景群像 | 跳过 |
| 简化提取（功能角色） | 出场2-3次但只承担单一功能 | `{功能标签}` + `{首次出场章节}` |
| 完整提取 | 出场3次以上，或有台词且推动剧情，或与主角有直接互动 | 完整档案 |

**别名归类**：同一角色不同称呼合并为同一实体。主条目用首次正式名，别名列于 `aliases: [别名1, 别名2]`。

### 深度 阶段2：情节点粒度

每章目标 10-15 个。一个情节点 = 一个可独立理解的叙事单元。

- 太粗：「主角和反派打了一架」 -> 丢失细节
- 太细：「主角举起右手」 -> 过度碎片化
- 恰当：「主角以三招击败反派，围观者震惊」 -> 完整戏剧事件

每章10-15个 -> 每条剧情75-225个，即每条剧情覆盖5-15章。

### 深度 阶段3：孤立情节兜底

（已归类 = 剧情条内 + 孤立归入；散落情节不计）

目标 85%-95%

输出：总{N} | 已归类{M}（剧情条{P} + 孤立{Q}） | 散落{R} | 覆盖率{X%}

### 错误处理表

| 场景 | 识别方式 | 处理策略 |
|------|----------|----------|
| 章节识别失败 | 无法匹配分隔符 | 提示用户确认格式；支持自定义正则 |
| 分块中断 | 会话超时/用户中断 | 读_progress.md断点，从当前块起始恢复 |
| 聚合质量不达标 | 覆盖率<85%或重叠率>35% | 孤立情节二次分类；放宽阈值至0.5重试 |
| 角色合并冲突 | 同一角色拆为多实体 | 记录待确认列表；提示用户确认 |
| 输出目录冲突 | 目录已有文件 | 追加不覆盖；冲突文件标记 `[重新分析]` |

### 质量阈值检查清单

阶段3-4完成前自检（阈值定义见 material-decomposition.md）：

- [ ] **置信度** >= 0.85（不达标 -> 标记「待复核」）
- [ ] **覆盖率** 85%-95%（<85% -> 二次分类；>95% -> 复核边界）
- [ ] **重叠率** <= 35%（>35% -> 提示边界模糊，建议合并）
- [ ] **角色合并**：候选合并已确认或标记待确认
- [ ] **散落情节**：散落情节附录已创建
- [ ] **角色分级**：有意义的角色已分级
