# 章节执行写作约束清单

## 强制合规规则

### 1. 字数控制
- 每章字数严格控制在融合指南 target_word_count 定义的区间
- 误差容忍度：±5%
- 超出时：删减非关键描写；不足时：扩展感官细节或对话

### 2. 风格指纹100%执行
- linguistic_ruleset 中的句长分布为硬约束
- dialogue_implementation 中的对话占比为硬约束
- descriptive_standards 中的感官优先级为硬约束
- 每章写完必须内部自检

### 3. 状态一致性
- 所有角色行动必须基于世界状态快照
- 禁止凭空创造未登记要素
- 禁止：角色在"昏迷"状态时对话
- 禁止：使用 inventory 未记录的道具
- 禁止：违反 established_rules 中的世界观定律

### 4. 细纲忠实执行
- 每个 beat 必须完整呈现
- beat_type 和 content 不得擅自更改
- 允许：在 beat 框架内扩展细节/丰富对话
- 不得改变 beat 功能定位

### 5. 伏笔机制
- foreshadowing_planted 标记的伏笔必须植入
- foreshadowing_resolved 标记的伏笔必须回收
- 每处伏笔需在 YAML 增量包中记录

## 禁止模式

| 模式 | 示例 | 替代方案 |
|------|------|----------|
| AI说明文腔调 | "值得一提的是" | 直接叙述 |
| 过度阐释 | 行为后解释动机 | 让行为自己说话 |
| 机械转场 | "与此同时" | 直接切入新场景 |
| 情绪贴标签 | "他很愤怒" | 生理反应showing |
| 网文烂梗 | "嘴角上扬" | 具体微表情 |
| 信息倾倒 | 单段>3新角色 | 分散引入 |

## 质量门控

1. POV一致性：逐段审查感知范围
2. 对话占比：目标±10%区间
3. signature_words：每章达标
4. 节奏符合性：段落长度/单句段频率
